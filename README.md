## Install

```bash
git clone https://github.com/fredrir/dotfiles ~/dotfiles
cd ~/dotfiles
./setup.sh
```

`./setup.sh` without arguments opens an interactive picker for the environment (and, when the environment includes a group with machine overrides, a second picker for the override). Pass a profile to skip the picker:

```bash
./setup.sh arch-linux/kde
./setup.sh arch-linux/hyprland
./setup.sh macos
```

## VPS / headless server

```bash
bash -c "$(curl -fsSL https://raw.githubusercontent.com/fredrir/dotfiles/main/bootstrap-vps.sh)"
```

Already cloned, or re-running

```bash
~/dotfiles/bootstrap-vps.sh
./setup.sh ubuntu/server
```

## The dotfile command

```bash
dotfile add waybar
dotfile add --description "Status bar" waybar
dotfile add --linux zsh/conf.d/11-linux-env
dotfile add --kde konsolerc
dotfile add --pkg zsh ~/.zshrc

dotfile link
dotfile link arch-linux/kde
dotfile link --override linux/hyprland=laptop
dotfile link -n

dotfile status
dotfile packages
dotfile format
```

`packages.dotfile` stores optional descriptions for tracked packages. `PACKAGES.md` is regenerated by `dotfile add`, `dotfile packages`, and the pre-commit hook.

## Adding a machine

1. Run `./setup.sh` and pick the environment under `environment/<os>/<desktop>`
2. If the environment includes a group with an `overrides/` directory (machine-specific config such as `linux/hyprland/overrides/desktop` and `linux/hyprland/overrides/laptop`), pick the override that matches the machine, or `none`

Both choices are saved in `~/.config/dotfile/`, so later `dotfile link` runs reuse them without prompting. Change the override any time with `dotfile link --override linux/hyprland=<name|none>`.
